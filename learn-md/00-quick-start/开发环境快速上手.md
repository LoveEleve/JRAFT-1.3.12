# 🚀 从 0 到 1 搭建 JRaft 开发环境（快速上手指南）

> **本篇目标**：在 30 分钟内跑通 JRaft 的第一个单元测试、第一次 Debug 核心路径，为后续源码阅读打下基础。

---

## 一、环境要求

| 项目 | 要求 | 说明 |
|------|------|------|
| JDK | **1.8+**（推荐 JDK 8 或 JDK 11） | JRaft 1.3.x 基于 JDK 8 编译 |
| Maven | **3.5+** | 用于项目构建 |
| IDE | IntelliJ IDEA（推荐） | 也支持 Eclipse |
| OS | Linux / macOS / Windows | Linux 体验最佳（RocksDB JNI） |
| 磁盘 | ≥ 2GB 空闲 | RocksDB 依赖需要编译本地库 |

---

## 二、获取源码 & 编译

### 2.1 克隆项目

```bash
git clone https://github.com/sofastack/sofa-jraft.git
cd sofa-jraft

# 切到文档分析的版本（1.3.12）
git checkout 1.3.12
```

### 2.2 编译

```bash
# 跳过测试，加速首次编译
mvn clean install -DskipTests -Dmaven.javadoc.skip=true

# 如果 RocksDB JNI 下载失败（国内网络问题），可以用阿里云镜像
# 在 ~/.m2/settings.xml 中配置：
# <mirror>
#   <id>aliyun</id>
#   <mirrorOf>central</mirrorOf>
#   <url>https://maven.aliyun.com/repository/central</url>
# </mirror>
```

**常见编译问题**：

| 问题 | 解决方案 |
|------|---------|
| `rocksdbjni` 下载超时 | 配置阿里云 Maven 镜像 |
| `tools.jar` 找不到 | 确保 `JAVA_HOME` 指向 JDK 而非 JRE |
| Windows 上 RocksDB 编译失败 | 使用 WSL 或 Docker |
| 内存不足 `OutOfMemoryError` | `export MAVEN_OPTS="-Xmx1024m"` |

## 三、IDEA 导入项目

### 3.1 导入

1. `File → Open` → 选择 `sofa-jraft` 根目录
2. 选择 `Import project from external model → Maven`
3. 等待 Maven 索引下载完毕（首次可能 5-10 分钟）

### 3.2 关键模块定位

在 IDEA 的 Project 视图中，你最关注的目录：

```
sofa-jraft/
├── jraft-core/src/main/java/com/alipay/sofa/jraft/
│   ├── core/          ← NodeImpl, Replicator, BallotBox 等核心类
│   ├── storage/       ← LogManager, RocksDBLogStorage 等存储类
│   ├── entity/        ← LogEntry, PeerId, Ballot 等实体类
│   ├── rpc/           ← RPC 处理器
│   └── option/        ← NodeOptions, RaftOptions 等配置类
│
├── jraft-core/src/test/java/
│   └── com/alipay/sofa/jraft/  ← 单元测试（Debug 入口）
│
├── jraft-example/     ← 使用示例（Counter、AtomicRange）
│   └── src/main/java/com/alipay/sofa/jraft/example/counter/
│       ├── CounterServer.java    ← 启动一个 Counter 节点
│       └── CounterClient.java    ← 客户端写入/读取
│
└── jraft-rheakv/      ← 分布式 KV 存储
```

### 3.3 IDEA 调试配置

1. 在 `NodeImpl` 上 `Ctrl+Click` 快速定位核心类
2. 建议在以下位置设置断点：
   - `NodeImpl.init()` — 节点启动入口
   - `NodeImpl.apply(Task)` — 写请求入口
   - `NodeImpl.handleRequestVoteRequest()` — 投票请求处理
   - `Replicator.sendEntries()` — 日志复制核心
   - `FSMCallerImpl.doCommitted()` — 状态机执行

## 四、跑通第一个单元测试

### 4.1 最简单的测试：NodeTest

```bash
# 命令行运行
mvn test -pl jraft-core -Dtest=com.alipay.sofa.jraft.core.NodeTest#testTripleNodes -DfailIfNoTests=false
```

或者在 IDEA 中：

1. 打开 `jraft-core/src/test/java/com/alipay/sofa/jraft/core/NodeTest.java`
2. 找到 `testTripleNodes` 方法
3. 点击方法左侧的绿色三角 → `Run`

**这个测试做了什么？**
- 创建 3 个 Raft 节点（同一进程，不同端口）
- 等待选举完成，验证有且仅有一个 Leader
- 发送写请求，验证数据一致性

### 4.2 Debug 选举流程

1. 在 `NodeImpl.handleElectionTimeout()` 设断点
2. 在 `NodeImpl.preVote()` 设断点
3. 在 `NodeImpl.electSelf()` 设断点
4. 在 `NodeImpl.becomeLeader()` 设断点
5. 以 Debug 模式运行 `testTripleNodes`

**你会看到的调用链**：

```
electionTimer 超时
  → handleElectionTimeout()
    → preVote()
      → 发送 RequestVoteRequest (isPreVote=true)
      → handlePreVoteResponse()
        → electSelf()
          → 发送 RequestVoteRequest (isPreVote=false)
          → handleRequestVoteResponse()
            → becomeLeader()
```

### 4.3 Debug 写请求流程

1. 在 `NodeImpl.apply(Task)` 设断点
2. 在 `LogManagerImpl.appendEntries()` 设断点
3. 在 `Replicator.sendEntries()` 设断点
4. 在 `BallotBox.commitAt()` 设断点
5. 在 `FSMCallerImpl.doCommitted()` 设断点
6. 运行带写操作的测试（如 `testTripleNodes`）

## 五、运行 Counter 示例（真实集群体验）

### 5.1 启动 3 节点集群

打开三个终端：

```bash
# 终端 1：节点 A
mvn exec:java -pl jraft-example \
  -Dexec.mainClass="com.alipay.sofa.jraft.example.counter.CounterServer" \
  -Dexec.args="counter 127.0.0.1:8081 127.0.0.1:8081,127.0.0.1:8082,127.0.0.1:8083 /tmp/jraft-counter/node1"

# 终端 2：节点 B
mvn exec:java -pl jraft-example \
  -Dexec.mainClass="com.alipay.sofa.jraft.example.counter.CounterServer" \
  -Dexec.args="counter 127.0.0.1:8082 127.0.0.1:8081,127.0.0.1:8082,127.0.0.1:8083 /tmp/jraft-counter/node2"

# 终端 3：节点 C
mvn exec:java -pl jraft-example \
  -Dexec.mainClass="com.alipay.sofa.jraft.example.counter.CounterServer" \
  -Dexec.args="counter 127.0.0.1:8083 127.0.0.1:8081,127.0.0.1:8082,127.0.0.1:8083 /tmp/jraft-counter/node3"
```

### 5.2 发送请求

```bash
mvn exec:java -pl jraft-example \
  -Dexec.mainClass="com.alipay.sofa.jraft.example.counter.CounterClient" \
  -Dexec.args="counter 127.0.0.1:8081,127.0.0.1:8082,127.0.0.1:8083"
```

### 5.3 实验

- **Kill Leader**：`Ctrl+C` 终端 1（假设节点 A 是 Leader），观察终端 2/3 的选举日志
- **恢复节点**：重新启动节点 A，观察它以 Follower 身份追赶日志
- **查看快照**：`ls /tmp/jraft-counter/node1/snapshot/` 查看快照文件

## 六、常用 Debug 技巧

### 6.1 开启详细日志

在 `jraft-core/src/test/resources/log4j2.xml` 中调整日志级别：

```xml
<!-- 开启 Raft 核心日志 -->
<Logger name="com.alipay.sofa.jraft" level="DEBUG"/>

<!-- 开启 Replicator 详细日志 -->
<Logger name="com.alipay.sofa.jraft.core.Replicator" level="DEBUG"/>
```

### 6.2 USR2 信号诊断（Linux/macOS）

```bash
# 找到 JRaft 进程 PID
jps | grep CounterServer

# 发送 USR2 信号，dump 所有节点状态
kill -USR2 <pid>

# 查看输出（在应用日志文件中）
tail -100 /tmp/jraft-counter/node1/logs/jraft.log
```

### 6.3 关键日志关键词

| 关键词 | 含义 |
|--------|------|
| `Node ... term changed` | term 变化，可能有选举 |
| `is_leader` | 选举结果 |
| `stepDown` | Leader 下台 |
| `Replicator ... send entries` | 日志复制 |
| `committedIndex` | 提交进度推进 |
| `Snapshot ... save` | 快照保存 |
| `onError` | 节点出错，进入 ERROR 状态 |

## 七、推荐阅读顺序

搭好环境后，按这个顺序逐篇阅读，效率最高：

```
01-overview（地图）
  → 02-node-lifecycle（启动流程 → Debug init()）
    → 03-leader-election（选举 → Debug preVote/electSelf）
      → 04-log-replication（日志复制 → Debug sendEntries）
        → 05-log-storage（RocksDB 存储）
          → 06-snapshot（快照 → Debug doSnapshot）
            → 07-state-machine（状态机 → Debug doCommitted）
              → 08-read-index（线性一致读）
                → 09-13（进阶模块，按需阅读）
                  → 15-raft-paper-vs-jraft（总结对照）
```

每读一个模块，建议：
1. **先读通俗解读**：建立直觉
2. **再读 README**：深入源码
3. **最后 Debug**：在对应断点处验证你的理解
